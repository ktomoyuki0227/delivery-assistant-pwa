# スマホでの動作確認ガイド

## 📱 スマホからアクセスする方法

### ステップ1: PCのIPアドレスを確認

開発サーバーを起動すると、以下のように表示されます:

```
- Local:   http://localhost:3000
- Network: http://192.168.x.x:3000
```

**Network**のURLをメモしてください。これがスマホからアクセスするURLです。

もし表示されない場合は、以下のコマンドでIPアドレスを確認:

#### Windows
```bash
ipconfig
```
`IPv4 アドレス`の値をメモ (例: `192.168.1.100`)

#### Mac/Linux
```bash
ifconfig | grep "inet "
```

### ステップ2: スマホとPCを同じWi-Fiに接続

- PCとスマホを**同じWi-Fiネットワーク**に接続してください
- 異なるネットワーク(例: PCが有線、スマホがWi-Fi)だとアクセスできません

### ステップ3: スマホのブラウザでアクセス

スマホのブラウザ(Safari/Chrome)で以下にアクセス:

```
http://[PCのIPアドレス]:3000
```

例: `http://192.168.1.100:3000`

### ステップ4: カメラ権限を許可

初回アクセス時、ブラウザがカメラへのアクセスを求めます:

- **iOS Safari**: 「許可」をタップ
- **Android Chrome**: 「許可」をタップ

## ⚠️ よくある問題と解決方法

### 問題1: スマホからアクセスできない

**原因**: ファイアウォールがポート3000をブロックしている

**解決策(Windows)**:
1. **Windowsセキュリティ** を開く
2. **ファイアウォールとネットワーク保護** → **詳細設定**
3. **受信の規則** → **新しい規則**
4. **ポート** → **TCP** → **特定のローカルポート: 3000**
5. **接続を許可する** → 名前を「Next.js Dev Server」に設定

### 問題2: カメラが起動しない(HTTP接続の場合)

**原因**: HTTPSでないとカメラが動作しないブラウザがある

**解決策**:
1. **開発環境**: `localhost`でアクセス(HTTPでも動作)
2. **本番環境**: Vercelにデプロイ(自動HTTPS)

### 問題3: 「このサイトにアクセスできません」エラー

**原因**: PCとスマホが異なるネットワークにいる

**解決策**:
1. PCとスマホを同じWi-Fiに接続
2. PCのIPアドレスが正しいか確認
3. 開発サーバーが起動しているか確認

## 🎯 推奨: Vercelにデプロイ(HTTPS対応)

スマホで完全にテストするには、HTTPS環境が必要です。

### クイックデプロイ

```bash
# Vercel CLIをインストール
npm install -g vercel

# デプロイ
vercel
```

デプロイ後、HTTPS対応のURLが発行されます:
```
https://your-app.vercel.app
```

このURLをスマホでアクセスすれば、カメラが完全に動作します!

## 📸 テスト手順

1. **ホーム画面**: アプリが正しく表示されるか確認
2. **カメラ画面**: カメラが起動するか確認
3. **撮影**: 何か撮影してみる
4. **テキスト確認**: OCR処理が動作するか確認(モックデータ)
5. **地図表示**: 地図が表示されるか確認
6. **ナビ開始**: Googleマップが起動するか確認
